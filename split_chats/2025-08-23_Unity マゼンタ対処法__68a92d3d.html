Unity マゼンタ対処法

Unityアセットストアでダウンロードしたアセットのマテリアルがマゼンタになっているときの対処法

Unity でアセットをインポートしたときに マテリアルやテクスチャがマゼンタ（ピンク） になるのは、基本的に「参照しているシェーダーが見つからない／動作しない」ことが原因です。いくつか原因と対処法を整理します👇

✅ 主な原因と対処法

1. シェーダーがプロジェクトに存在しない

  • アセットが独自シェーダーを使っているのに、そのシェーダーファイルが含まれていない場合。
  • 対処法
  • インポート時に Shaders フォルダをスキップしていないか確認。
  • .shader ファイルが存在するか確認し、なければ再インポート。

2. パイプラインの不一致

Unity では描画パイプラインが 3 種類あります：
- Built-in（従来の標準レンダリング）
- URP（Universal Render Pipeline）
- HDRP（High Definition Render Pipeline）

ダウンロードしたアセットが Built-in 用なのに、プロジェクトが URP/HDRP だとシェーダーが対応せず マゼンタ になります。

  • 対処法
  • プロジェクトのパイプラインを確認：
    Edit > Project Settings > Graphics / Quality
  • アセットに URP/HDRP 用のコンバーターが同梱されていれば、
    Assets > Upgrade Materials to URP/HDRP を実行。
  • なければ手動でマテリアルの Shader を Universal Render Pipeline/Lit や HDRP/Lit に変更。

3. シェーダーのコンパイルエラー

  • コンソールに Shader error が出ている場合。
  • 対処法
  • Console ウィンドウでエラーを確認し、必要ならシェーダーコードを修正。
  • 古いアセットの場合は Unity のバージョンに合わずエラーが出ていることもあるので、アセットストアのページで「対応 Unity バージョン」を確認。

4. テクスチャが欠損している

  • シェーダーはあるが、テクスチャがインポートされていない。
  • 対処法
  • マテリアルを選択して、Albedo などのスロットにテクスチャが割り当てられているか確認。
  • 無い場合はアセット内の Textures フォルダから手動で設定。

5. グラフィック設定やビルド対象の不一致

  • Mobile 用アセットを PC で使うと一部のシェーダーが動かないことも。
  • 対処法
  • マテリアルを簡易シェーダー（例: Unlit/Texture や URP/Lit）に置き換えて表示確認。

🔧 最終手段

  • すべてのマテリアルを一括変換
  • URP なら Edit > Render Pipeline > Universal Render Pipeline > Upgrade Project Materials to URP Materials
  • HDRP なら同様に HDRP 用メニューから変換

👉 まずは、
- コンソールエラーが出ていないか？
- プロジェクトが Built-in / URP / HDRP のどれなのか？

ここを確認すると解決が早いです。
